# v0.9 Validation Checklist

This checklist defines the repeatable validation scope before BootProfile
Switcher moves toward the initial stable release. The current validation target
is the configuration-driven runtime path. No real system-changing module should
be introduced as part of this checklist.

## Preconditions

Run validation from an elevated PowerShell session unless a step explicitly says
otherwise.

```powershell
cd D:\Git\ctreffe\agit-windows-bootprofile-switcher
```

The managed boot menu and startup hook should be installed when validating real
Mode A and Mode B startup behavior:

```text
install.cmd
install-configuration.cmd
install-startup-hook.cmd
```

The default machine-wide configuration path is:

```text
C:\ProgramData\BootProfileSwitcher\config\profiles.json
```

## Log Assertions

Use these fields in `logs/startup-profile.log` as the primary runtime
assertions:

```text
detected=
mode=
profileScriptExecuted=
modulesExecuted=
configurationValid=
profileConfigured=
dispatchSkippedReason=
```

Use `logs/module-actions.log` to confirm that the harmless validation module
actually ran when `modulesExecuted=validation-log`.

The post-v0.9 development version may also execute the temporary
`demo-system-marker` module for the v1.0.0 release demonstration. That module
writes a harmless marker to
`C:\ProgramData\BootProfileSwitcher\runtime\demo-current-profile.json`.

Command:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\Test-BootProfileConfigurationFixtures.ps1 -AsJson
```

Expected result:

```text
"passed": true
"failed": 0
```

Validated on 2026-06-28.


Validated on 2026-06-28.

## V3: Mode A Runtime Path

Boot into `BootProfile Switcher - Mode A`, then run:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\Invoke-BootProfileStartupHook.ps1
Get-Content -Tail 3 -Encoding UTF8 .\logs\startup-profile.log
Get-Content -Tail 3 -Encoding UTF8 .\logs\module-actions.log
```

Expected startup log assertions:

```text
detected=True
mode=A
configurationValid=True
profileConfigured=True
modulesExecuted=validation-log
profileScriptExecuted=False
dispatchSkippedReason=
```

Expected module log assertion:

```text
module=validation-log
mode=A
```

Validated on 2026-06-28.

## V8: Reinstall Safety

Run `install.cmd` when managed BootProfile Switcher entries already exist.

Expected result:

```text
Installer detects existing BootProfile Switcher entries.
Installer offers cleanup before creating fresh entries.
After installation, Get-BootProfileMenuStatus.ps1 reports exactly one Mode A entry and one Mode B entry.
```

Verification command:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\Get-BootProfileMenuStatus.ps1
```

Validated on 2026-06-28.
