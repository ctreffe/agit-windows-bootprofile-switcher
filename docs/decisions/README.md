# Decision Records

This directory contains durable BootProfile Switcher decisions.

- `ADR` records architecture, configuration, lifecycle, deployment, security,
  fixture or generated-artifact decisions.
- `PDR` records project scope, roadmap, privacy, repository or governance
  decisions.
- `DDR` records durable user-documentation structure, terminology, examples or
  documentation-QA decisions.

Use a stable prefix, four-digit number and short kebab-case title. Start new
records from the matching `*-0000-template.md` file, replace all placeholders
and record status and date. Decision records explain context, decision,
rationale, consequences, alternatives and follow-up; they are not required for
minor implementation choices.
